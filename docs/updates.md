# Updates

这份文档用来说明：**当前版本相对上一版 `README / installation / usage` 文档，实际已经发生了哪些变化。**

它不是完整使用手册，主要作用是帮你快速回看这段时间到底改了什么，哪些点已经和旧文档不一样了。

---

## 总体方向变化

最重要的一条是：**Sprout 现在已经不再依赖浏览器域名识别，也不再依赖 ActivityWatch。**

上一版文档里还有不少内容默认把 Sprout 当成“桌面窗口 + 浏览器标签页”的混合工具来看待；现在实际产品边界已经收缩成：

- Sprout 负责桌面窗口层面的拦截
- 浏览器里的具体网页内容交给浏览器插件处理

也就是说，浏览器仍然可以作为一个普通窗口加入白名单或通过分类放行，但 Sprout 不再区分浏览器里的具体网站。

---

## 功能层面的新增

### 1. 正计时模式

现在除了原来的倒计时，还支持正计时模式。

- 倒计时：设置一个时长，到点自动完成
- 正计时：从 `00:00` 开始往上计时，只能手动结束

这是一条真正的新功能，应该视为 1.1.0 级别的更新，不是小修小补。

### 2. 完成专注提醒

倒计时自然结束后，现在会触发 Windows 系统通知，并伴随系统提示音。

也就是说，用户不需要一直盯着计时器看，专注完成时会收到系统级提醒。

### 3. 打包版可用

现在已经能稳定生成并运行：

- `win-unpacked` 便携目录
- Windows 安装包 `Sprout Setup 1.1.0.exe`

之前安装包“有进程但没界面”的问题已经修过，主进程不再依赖外部 Node 启动 guardian。

## 行为逻辑上的修复

### 1. 分类规则同步修复

之前存在一个很坑的问题：你编辑了分类规则，但本轮已选分类里还是旧快照，启动会话后 guardian 实际跑的是旧规则。

这个问题已经修掉了。现在分类规则库和本轮已选分类会同步对齐，不会再出现“界面看起来改了，实际运行还是老 pattern”的情况。

### 2. 重复违规处理修复

之前同一个违规窗口会在时间线里刷出很多条重复记录，后来为了压重复记录又一度出现“第二次违规不再拦截”的 bug。

现在的逻辑是：

- 同一个窗口持续违规时，仍然继续拦截
- 但不会反复新增一堆重复日志

### 3. 违规后不再强行恢复最近允许窗口

之前的实现会在最小化违规窗口后，主动把最近一次允许窗口拉到前台。这个行为会带来副作用，比如你手动最小化的白名单窗口被程序又抢回来。

现在已经改成：

- 只最小化当前违规窗口
- 然后让 Windows 自己决定前台切到谁

Sprout 不再替用户决定“接下来该看哪个窗口”。

### 4. 自身窗口识别更稳

Sprout 自己的窗口现在优先按 `processId === process.pid` 识别，不再主要依赖标题或路径里有没有 `sprout`。

这让开发态、打包态、自身 Electron 壳窗口的放行都稳定了很多。

### 5. Snipaste 白名单

系统安全白名单里已经补了 `Snipaste.exe` 和一些相关标题关键词，避免截图/贴图工具被误拦截。

## UI 层面的更新

### 1. 小窗口布局重做

小窗口下的主界面已经做过一轮收紧：

- 顶部更紧凑
- 规则摘要和当前上下文布局更合理
- 历史、设置入口回到图标按钮

### 2. 历史抽屉重做

历史抽屉现在已经改成：

- 顶部统一头部
- 操作按钮缩成一行小按钮
- 只显示最近三天
- 去掉中间“会话索引”列
- 右侧 Markdown 预览区更宽
- 日期卡支持直接打开实际 Markdown 文件

### 3. 设置抽屉重做

设置抽屉也已经统一到和历史抽屉接近的风格：

- 头部结构统一
- 左窄右宽
- 当前上下文原始对象预览区更宽
- 系统安全白名单单独放在下方整宽区域

### 4. 品牌文案调整

左上角的大标题已经改成：

- `生长空间`
- `杂草出现，即刻清除。`

这和之前“专注白名单”的表述已经不一样了。

## 文档层面应该同步的地方

如果你回头要继续整理 `README / installation / usage`，有几条一定要和旧版本切开：

1. 不再写 ActivityWatch、域名规则、浏览器标签页识别
2. 要写清楚浏览器内容交给插件处理
3. 要补上正计时模式
4. 要补上完成专注提醒
5. 要说明现在已经有可直接安装运行的 Windows 安装包

---

## 当前版本更接近什么

和最早那版相比，现在的 Sprout 更像一个：

**桌面窗口专注工具 + 托盘驻留 + 历史记录 + 倒计时/正计时双模式**

而不是一个浏览器域名识别工具。

这个边界现在清楚很多，也稳定很多。
